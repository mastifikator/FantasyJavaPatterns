Посредник — это поведенческий паттерн, который упрощает коммуникацию между компонентами системы.

Посредник убирает прямые связи между отдельными компонентами, заставляя их общаться друг с другом через себя.

В данном примере реализована многопоточная битва между Орками, Людьми и Нежитью
Можете поиграться меняя значения здоровья и повреждений в методе Main

Посредником в данном случае является класс DamageMediator который принимает 2 объекта наследника абстрактного класса Unit
И реализует логику битвы между двумя объектами
Данный прием позволяет не захламлять основные классы Юнитов лишними поведенческими методами
