Посетитель — это поведенческий паттерн, который позволяет добавить новую операцию для целой иерархии классов, не изменяя код этих классов.
Посетитель имеет смысл реализовывать когда нету доступа к коду классов которые он собирается посетить

В данном примере реализован сценарий в котором Visitor наносит визит лидерам разных рас и меняет их поведение
для этого в Visitor реализованы методы для каждого класса
